UAV Forecast API Documentation

Version: v0.1 (beta)

Last updated: 2025-06-18

This document describes the API for UAV Forecast, which provides weather and satellite forecast information for drones.

For support and other inquiries, please contact api@UAVForecast.com.

Overview

The base endpoint for the API is:

POST https://www.uavforecast.com/api/v1/forecast

The API accepts a JSON request body, defined by ForecastRequestV1, containing the desired location and forecast parameters. The response is a JSON object, defined by ForecastResponseV1, which includes current conditions, forecast data, and cost information.

Your API key should be supplied in the Authorization header (see below). If you do not have an API key yet, please contact api@UAVForecast.com.

Warning: It is your responsibility to keep your API key secret. Do not include your API key in any binaries or other files distributed to the public. This includes API keys compiled into a binary, obfuscated or otherwise.

API Cost

Every response returns a cost field indicating the cost incurred by the request. The API provides a mechanism to determine the cost of an API request without actually incurring that cost; see cost_only below.

The cost of each API request currently depends on three factors:

  • The number of hours of forecast requested.
  • The maximum wind altitude.
  • The number of wind altitude levels.

Note that the forecast engine produces forecasts at key altitudes, and the cost depends on which of these key altitudes are used. Therefore, you may find the cost does not always increase when requesting additional altitudes between these key levels.

Here are some example costs:

Forecast Hours Wind Altitudes (m) Cost
0 10 $0.20
24 10 $0.22
7 * 24 = 168 10, 30, 80, 200, 500 $0.31
15 * 24 = 360 10, 30, 80, 200, 500, 1000, 1500 $0.41

Request Format

Endpoint

POST https://www.uavforecast.com/api/v1/forecast

Please note that for security, all requests must be made via HTTPS.

Headers

Header Description
Authorization Bearer <api-key>
Content-Type application/json

Body

The body must be a JSON object described by the schema ForecastRequestV1.

Response Format

Success Response

A successful request will return a JSON object following the schema ForecastResponseV1.

Note that requests for many forecast hours and wind altitudes may produce large JSON responses. To improve latency, the API supports gzip compression, which will be used when the Accept-Encoding header in the request includes gzip. We recommend using a fast JSON deserialization library on the client side to improve application latency; for example, in Python, we recommend using orjson instead of Python's built-in json library.

Cost-Only Requests

When cost_only is set to true in the request, the API will respond with CostOnlyResponseV1. If and only if a cost would have otherwise been incurred by the request, the response will contain a cost field of type CurrencyAmount equal to that cost. Cost-only requests do not consume any API balance.

Error Responses

In the event of errors (e.g., missing parameters, invalid API key), the API will return appropriate HTTP status codes (4xx for client errors, 5xx for server errors) and a JSON payload that may contain details about the error. For instance:

{
  "detail": "Invalid API key"
}

Schemas

ForecastRequestV1

Field Type Description
cost_only boolean (optional) If set to true, returns the cost of the request, without returning real data or incurring a charge.
lat float The latitude of the location, in degrees, e.g. 21.409.
lon float The longitude of the location, in degrees, e.g. -157.961.
forecast_hours integer (optional) The number of hours ahead to forecast. Defaults to 0.
wind_altitudes_m array of integer (optional) An array altitudes (in meters) at which to request the wind forecast. The valid range is 10m to 5000m. Note that an altitude of 10m is considered ground level. The response will include wind data at each altitude requested. If you need wind data at ground level, be sure to include 10m in the list. Defaults to [10], in which case the response will include wind data for the ground level only.
use_realtime_precip boolean (optional) If true, use real-time precipitation data. Defaults to true.
include_gps boolean (optional) If true, include GPS satellite data. Defaults to true.
include_glonass boolean (optional) If true, include GLONASS satellite data. Defaults to true.
include_galileo boolean (optional) If true, include Galileo satellite data. Defaults to true.
include_beidou boolean (optional) If true, include Beidou satellite data. Defaults to true.
gps_elevation_mask integer (optional) The GPS elevation mask, in degrees. Satellites below this elevation angle are considered invisible. Defaults to 0.
kp_source string (optional) The source for the Kp index forecast. Possible values:
- auto (default)
- noaa to use NOAA as the source.

ForecastResponseV1

Field Type Description
cost CurrencyAmount (optional) The actual cost of the request.
current ForecastRowV1 Current conditions.
days array of ForecastDayV1 A list of forecast days. The first entry is always the current day.
elevation_m float The elevation of the forecast location, in meters.
timezone string The timezone name, e.g. America/New_York.
lat float The latitude of the location, in degrees.
lon float The longitude of the location, in degrees.

CostOnlyResponseV1

Field Type Description
cost CurrencyAmount (optional) The cost that would have been incurred by the request.
cost_only boolean Indicates that this is a cost-only response. Always set to true.

ForecastDayV1

Field Type Description
date string The local date in ISO 8601 format, e.g. 2024-12-31.
midnight TimeV1 The time at local midnight.
sunrise TimeV1 (optional) The time of local sunrise. If sunrise does not occur on the given date (e.g. at high latitudes), this field may be absent.
solar_noon TimeV1 The time of local solar noon.
sunset TimeV1 (optional) The time of local sunset. If sunset does not occur on the given date (e.g. at high latitudes), this field may be absent.
rows array of ForecastRowV1 A list of forecast rows for each hour remaining in the day. The time fields of the rows always have the minute set to zero. The list will be empty if there are no future hours left in the day.

ForecastRowV1

Field Type Description
time TimeV1 Time of the forecast.
wind_profile array of WindProfileV1 Wind profile at different altitudes.
temp_c float Temperature, in degrees Celsius.
wind_chill_c float Wind chill temperature, in degrees Celsius.
precip_prob_pct float Probability of precipitation, as a percentage from 0 to 100.
precip_intensity_mmh float Precipitation intensity, in millimeters of liquid water per hour.
precip_type string (optional) Type of precipitation, one of: rain, snow, sleet, or hail.
cloud_cover_pct float Cloud cover, as a percentage from 0 to 100.
visibility_m float Visibility, in meters.
humidity_pct float Relative humidity, as a percentage from 0 to 100.
cloudbase_m float (optional) Cloud base altitude, in meters. If there are no clouds, this field may be absent.
sats GpsSatellitesV1 Instantaneous GPS satellite data.
weather_icon string Weather icon code, one of: clear-day, clear-night, rain, snow, sleet, wind, fog, cloudy, partly-cloudy-day, or partly-cloudy-night.
density_altitude_m float Density altitude, in meters.
pressure_msl_hpa float Mean sea level pressure, in hectopascals (hPa).
kp float The Kp index.

WindProfileV1

Field Type Description
altitude_m integer (optional) Altitude, in meters. If absent, or set to 10m, indicates surface (ground) level data.
wind_speed_ms float Wind speed, in meters per second.
wind_bearing_deg float (optional) Wind bearing, in degrees, in the range 0 to 360. The bearing points in the direction the wind is blowing from. For example, if the wind is blowing from the South, the bearing will be 180 degrees. If the wind speed is zero, this field may be absent.
gust_speed_ms float Gust speed, in meters per second.
gust_bearing_deg float (optional) Gust bearing, in degrees, in the range 0 to 360. The bearing points in the direction the wind is blowing from. For example, if the gust is blowing from the South, the bearing will be 180 degrees. If the gust speed is zero, this field may be absent.
t_c float Temperature, in degrees Celsius.

GpsSatellitesV1

Field Type Description
gps SatelliteCountV1 (optional) GPS satellite count.
glonass SatelliteCountV1 (optional) GLONASS satellite count.
galileo SatelliteCountV1 (optional) Galileo satellite count.
beidou SatelliteCountV1 (optional) Beidou satellite count.

SatelliteCountV1

Field Type Description
count integer The number of satellites visible.
kp_count float The expected number of satellites locked, taking the Kp index and other factors into consideration.

TimeV1

Field Type Description
utc string UTC time in ISO 8601 format, e.g. 2025-01-10T00:00:00Z.
local string Local time in ISO 8601 format, e.g. 2025-01-09T14:00:00-10:00.
epoch_s integer Unix epoch time in seconds, e.g. 1736467200.

CurrencyAmount

Field Type Description
amount float The amount as a decimal, e.g. 0.39.
currency string The currency code, e.g. USD.

Example Python Code

import requests

API_URL = "https://www.uavforecast.com/api/v1/forecast"
API_KEY = "test-s2SYt1vqS4BNKfLqffyDCbY6Z1Yo2Kmi"

# Request payload
payload = {
    "lat": 21.409,  # Honolulu, Hawaii
    "lon": -157.961,
    "forecast_hours": 24,
    "wind_altitudes_m": [10, 100, 200],
    "gps_elevation_mask": 5,
}

# HTTP headers
headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"}

# Send request
response = requests.post(API_URL, json=payload, headers=headers)

# Handle response
if response.status_code == 200:
    data = response.json()
    print("Forecast received:")
    print(data)
else:
    status_code = response.status_code
    message = response.json()["detail"]
    print("Error response:")
    print(f"Status code: {status_code}")
    print(f"Message: {message}")
